# BRIDGE PATTERN (STRUCTURAL)

## Intent
The Bridge pattern is designed to decouple an abstraction from its implementation so
that the two can vary independently. It is used to separate a high-level logic
hierarchy from a platform-specific implementation hierarchy.

## The Problem
When a class has multiple dimensions of variability (e.g., different types of shapes
and different ways to draw them), using standard inheritance leads to a "Cartesian
Product" explosion of subclasses. Adding one new shape or one new platform requires
creating a specific class for every possible combination, making the codebase rigid
and unmanageable.

## The Solution
Instead of a single, deep inheritance tree, we split the code into two independent
dimensions:
1. **Abstraction**: The high-level control logic (e.g., what the object is).
2. **Implementer**: The specific technical execution (e.g., how the object is rendered).

The Abstraction holds a reference (the "Bridge") to an Implementer object, allowing
the two to grow independently.

## Our Examples
This directory illustrates the two most important ways the Bridge pattern is applied
in the C++ industry:

1. **Bridge_GoF/**:
The classic architectural implementation. It demonstrates how to decouple logical
shapes (Circle, Square) from rendering platforms (Vector, Raster). This is the
standard approach to avoid class explosion.

2. **Bridge_Impl/**:
The "Pimpl" (Pointer to Implementation) idiom. This is a C++-specific optimization
of the Bridge pattern used as a "compilation firewall". It decouples the class
interface from its internal members to speed up compilation times and hide
implementation secrets.

## Key Benefits
- **Independence**: You can modify the abstraction without touching the
- **implementation and vice versa.  Improved Compilation**: Specifically with the
- **Pimpl idiom**: changes in private members do not trigger full project
- **recompilations.  Runtime Flexibility**: The implementation can be swapped
- **dynamically**: at runtime by simply reassigning the bridge pointer.
